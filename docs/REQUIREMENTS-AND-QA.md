## 1. Original user brief (verbatim)

> I need you to create a static web site ready to be deployed to github pages (github.io).
> The site will be in Ukrainian language only. So you can use HTML, CSS and JS for dynamic actions and visual effects.
> The site should consist of 4 main sections: `Домашня сторінка`, `Усі матеріали`, `Рубрики`, `Допомога притулкам`, `Про нас`.
> All the necessary materials for the site are collected in the project's root `sources` directory.
> Home page's Hero text is in the `sources/Домашня сторінка/Домашня сторінка.html` doc.
> `Усі матеріали` page is a list of articles with their titles and preview pictures. Each article item should lead to a separate article page. The list of needed articles is in the `sources/Усі матеріали` folder. Article names are H1 in each HTML file of the `sources/Усі матеріали` folder.
> `Рубрики` is a dropdown menu with such options: `Художній репортаж` (художній репортаж), `Інтервʼю` (інтервʼю), `Портретні нариси` (портрети), `Огляди` (екоцид, тварини в професії), where in the parenthesis are subfolder names of `sources/Усі матеріали` to the articles. Thus, both `Усі матеріали` and `Рубрики`sections in the end forward to the same article links. Each article page is accessible from `Усі матеріали` or from `Рубрики`.
> `Допомога притулкам` page is a list of links to external web sites. The list can be found in `sources/Допомога притулкам/Допомога притулкам.txt`.
> `Про нас` page contains the text from `sources/Про нас/Про нас.html` and a decorations up to you.
> When using materials from the `sources` folder, you should use texts and attached pictures. You are not allowed to change texts even if you see any lingual mistakes in them, some mistakes are made intentionally. You can reuse HTML tags how you'd like, but please don't use the CSS styles and classes from those files (those CSS were generated by MS Word and are not needed artifacts).
> The overall site's design and design of each page should be implemented by you. Use the best and latest design practices.
> When creating HTML, CSS and JS, use the latest standards. No need to support old browsers. If there are native HTML and CSS alternative to the same JS functionality, use the latest native standards if they don't create design regression.
> In this task you don't write any code, but only create detailed documentation which should be held out to AI agents of further sessions. The documentation should be kept in the `docs` folder, and contain a very detailed PRD, plans and tasks. The docs should be so detailed so they must not loose any part of this initial requirements and any of our further conversation: all this text and results of every your Q&A session should be included.
> Requirements and tasks related to the visual design and look'n'feel should also be present there.

### Standing constraints extracted from the brief (binding on all future work)

- **Static site**, deployable to **GitHub Pages**. HTML/CSS/JS only.
- **Ukrainian language only.** No language switcher.
- Sections named exactly: `Домашня сторінка`, `Усі матеріали`, `Рубрики`, `Допомога притулкам`, `Про нас`.
- **All content comes from `sources/`.** Use the texts and the attached pictures.
- **Texts must NOT be changed** — including apparent spelling/grammar mistakes (some are intentional).
- **Do NOT reuse the MS-Word CSS styles/classes** from the source files. HTML tags may be reused freely.
- **Design is the implementer's responsibility** — use best/latest practices.
- **Use latest web standards**; no legacy-browser support; **prefer native HTML/CSS over JS** when a
  native alternative exists and causes no design regression.
- This task = **documentation only**, no site code.

## 2. Clarifying Q&A (every question, options, and chosen answer)

Format: **Q** = question · options listed · **✔ Answer** = what the user chose.

### Q1 — Navigation structure
The brief says "4 main sections" but lists 5. How is the top nav structured?
- *5 nav items, Рубрики = dropdown* — Home | Усі матеріали | Рубрики (dropdown of 4 rubrics) | Допомога притулкам | Про нас.
- *Рубрики is its own landing page too.*

**✔ Answer: 5 nav items, Рубрики = dropdown.** "4 main sections" was a slip; there are 5 nav entries
and Рубрики is a dropdown, not a standalone page.

### Q2 — Rubric routing
When a rubric is picked from the dropdown, where does it lead? (Художній репортаж, Інтервʼю,
Портретні нариси have 1 article each; Огляди has 2.)
- *Always to a filtered list* — every rubric → an Усі-матеріали-style grid filtered to that rubric.
- *Smart: single→article, multi→list.*
- *All rubrics → anchor on Усі матеріали.*

**✔ Answer: Always to a filtered list.** Even single-article rubrics show a 1-card list. Consistent and
scalable as more articles are added.

### Q3 — Build approach
How much tooling for ~9 pages?
- *Plain static, no build step* — hand-authored files; an `articles` data model drives the grid and
  rubric filtering client-side; deploy by pushing a branch.
- *Static site generator (Astro/Eleventy).*
- *Plain static + tiny build for images only.*

**✔ Answer: Plain static, no build step.**

### Q4 — File / URL naming
Sources have Cyrillic names. What naming for built files/URLs/assets?
- *ASCII slugs (transliterated)* — clean Latin slugs; visible UI text stays Ukrainian; docs include a
  slug map.
- *Keep Cyrillic filenames.*

**✔ Answer: ASCII slugs (transliterated).**

### Q5 — GitHub Pages base path
Project site serves at `https://kirea.github.io/kotopes/` (a `/kotopes/` sub-path).
- *Relative paths, works at any sub-path* — all links/assets relative (`./`, `../`); deploy from a
  branch via Settings → Pages.
- *Assume root + custom domain later.*
- *Deploy via GitHub Actions to Pages.*

**✔ Answer: Relative paths, works at any sub-path.**

### Q6 — Design mood
Overall visual direction?
- *Editorial documentary.*
- *Warm & hopeful* — softer, humane shelter-charity feel; warm palette; rounded cards; friendly sans;
  emphasis on compassion and the help CTA.
- *Solemn wartime memorial.*
- *Modern minimal.*

**✔ Answer: Warm & hopeful.**

### Q7 — Palette
Which warm palette? (All keep UA blue/yellow as sparing CTA accent; all support dark mode.)
- *Sand + terracotta.*
- *Peach + sage.*
- *Honey + teal* — warm honey/ivory base, deep teal (#2A6F6B) accents, amber highlights; teal pairs
  with the yellow accent; warm but a touch more editorial/sophisticated.

**✔ Answer: Honey + teal.**

### Q8 — Typography
Type system (must have full Ukrainian Cyrillic incl. `ʼ`)?
- *Serif headings + sans body* — warm humanist serif headlines (Lora / PT Serif) + clean sans body
  (Inter / Onest). Classic long-read editorial warmth.
- *All-sans humanist.*
- *e-Ukraine (official UA font).*

**✔ Answer: Serif headings + sans body.**

### Q9 — Article images
How are inline article photos presented?
- *Inline in original positions + lightbox* — each photo kept where it sits in the source, wrapped in
  `<figure>` (with caption if present); click opens a native `<dialog>` lightbox; lazy-loaded.
- *Inline only, no lightbox.*
- *Gallery at the bottom.*

**✔ Answer: Inline in original positions + lightbox.**

### Q10 — Article cards (grid + filtered lists)
What do cards carry?
- *Title + image + rubric tag + excerpt* — preview photo, H1 title, rubric chip, and a 1–2 line
  excerpt taken verbatim from the article's opening sentence.
- *Title + image + rubric tag.*
- *Title + image only.*

**✔ Answer: Title + image + rubric tag + excerpt.**

### Q11 — Site name / branding
Header wordmark and `<title>`?
- *«Своїх не кидають» as the brand.*
- *A short project name.*
- *Text wordmark + simple paw/icon mark.*

**✔ Answer (custom): simple paw/icon mark + «Котопес».** Wordmark = **«Котопес»** plus a simple inline
paw SVG mark. The slogan **«Своїх не кидають» is reserved for the home hero only.**

### Q12 — Home page composition
Sections beyond the hero?
- *Hero + featured + rubrics + help CTA + about teaser* — Hero → «Рекомендовані матеріали» (3–4 cards)
  → rubric shortcut tiles (4) → «Допомога притулкам» CTA band → short «Про нас» teaser.
- *Hero + featured articles only.*
- *Hero + featured + help CTA.*

**✔ Answer: Hero + featured + rubrics + help CTA + about teaser.**

### Q13 — Допомога притулкам page
Source is only 3 raw URLs (no titles/descriptions). How to present?
- *Cards with org name + short blurb I write* — org name derived from the site (HAU / Домівка /
  Happy Paw), a 1-line factual Ukrainian description I write (additive, must be accurate), and a
  «Перейти на сайт ↗» button (new tab).
- *Cards with org name only.*
- *Plain styled link list.*

**✔ Answer: Cards with org name + short blurb I write.**

### Q14 — Про нас page
Composition? ("decorations up to you")
- *Editorial mission layout* — source text as a centered large-type mission statement with a warm
  backdrop, one line treated as a pull-quote, and a closing help CTA. Text verbatim, no invented facts.
- *Mission text + project facts.*
- *Simple centered text.*

**✔ Answer: Editorial mission layout.**

### Q15 — Non-functional / polish (multi-select)
Which to mandate?
- *Light + dark mode toggle.*
- *Subtle motion + reduced-motion.*
- *Accessibility (WCAG 2.2 AA).*
- *SEO + social meta + favicon.*

**✔ Answer (custom multi): Subtle motion + reduced-motion · Accessibility (WCAG 2.2 AA) · SEO + social
meta + favicon.** **Dark-mode toggle was NOT selected → single light theme.** User added a note:
> "also, for the home page design, you can generate your own pictures. the design should be visually
> appealing and eye-catchy"

### Q16 — Generated imagery direction
Style and placement for generated/custom pictures? (Article pages still use only source photos.)
- *Warm photographic, home + decor only* — photoreal warm-toned images of animals/rescue/hope
  (Honey+teal-consistent) for the home hero background and decorative section bands only; committed as
  optimized WebP.
- *Illustrated / editorial illustration.*
- *Minimal — CSS/SVG only, no raster gen.*

**✔ Answer: Warm photographic, home + decor only.**

### Q17 — Docs deliverable structure
How to structure `docs/`?
- *Multi-file docs set* — `PRD.md`, `DESIGN.md`, `IMPLEMENTATION-PLAN.md`, `TASKS.md`,
  `REQUIREMENTS-AND-QA.md`, plus `README.md` index.
- *Single comprehensive doc.*
- *Multi-file + per-page content specs.*

**✔ Answer: Multi-file docs set.**
